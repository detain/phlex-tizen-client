# Signing Guide

This document describes how to sign the Phlix Tizen client for deployment to Samsung Tizen TV devices.

## Why Signing Is Required

Samsung Tizen TVs only accept applications that have been cryptographically signed with a valid Tizen certificate. The signature proves the app was built by an authorized developer and has not been tampered with after signing. Without a valid signature, the `.wgt` package cannot be installed on physical TV hardware, even via Tizen Studio's device manager.

---

## Required Credentials

| Credential | Purpose |
|---|---|
| **Tizen SDK Certificate** (a profile `.xml` file generated by Tizen Studio) | Authorizes the app to run on Tizen TV devices |
| **Author Certificate** (`.p12`) | The actual signing key embedded in the certificate |
| **Author Certificate Password** | Protects the private key in the `.p12` file |

### Certificate Types

| Certificate | Use Case |
|---|---|
| **Author Certificate** (Tizen) | Signs all `.wgt` packages for physical Tizen TV deployment |
| **Distributor Certificate** (Tizen) | Optional second signing layer for store submission |
| **Samsung Certificate** (Samsung-proprietary) | Required in addition to the Tizen author certificate for Samsung TV store submission |

## Creating Tizen Certificates

### Prerequisites

- **Tizen Studio** installed ([developer.tizen.org](https://developer.tizen.org/development/tizen-studio/))
- A Samsung Account registered at [developer.samsung.com](https://developer.samsung.com) (required for Samsung TV certificates)

### Step 1: Generate an Author Certificate Using Tizen Studio

1. Open **Tizen Studio**
2. Go to **Certificate Manager** (Tools → Certificate Manager)
3. Click the **+** button to create a new certificate profile
4. Select **Tizen** → **TV** (or the appropriate profile for your target device)
5. Choose **Create a new author certificate**
6. Fill in the required fields:
   - **Distributor Name**: Your organization name
   - **Distributor Organization**: Your company/organization
   - **Password**: A strong password for the certificate
7. Tizen Studio generates an author certificate and saves it to your local certificate store

The certificate is stored at:
- **Windows**: `C:\Users\<username>\tizen-sdk-data\keystore\author.p12`
- **macOS**: `/Users/<username>/tizen-sdk-data/keystore/author.p12`

### Step 2: Register a Samsung Certificate (For Samsung TV Store Submission)

If you plan to submit to the Samsung Apps TV Store:

1. Go to [developer.samsung.com](https://developer.samsung.com) → My Page → Smart TV Apps
2. Register as a Samsung TV developer
3. In Tizen Studio, open **Certificate Manager** → **Samsung** tab
4. Select **Create Samsung Certificate** and log in with your Samsung Account
5. Generate the Samsung certificate and associate it with your author certificate

> **Note**: The Samsung certificate is device-specific. If you generate it on a different machine, you must re-register the device in the Samsung developer portal.

## Step-by-Step Manual Signing

The typical workflow is: build the web app → package it → sign the `.wgt` file.

### Step 1: Build the Web App

```bash
npm install
npm run build
```

The production bundle is output to `dist/`.

### Step 2: Package the App

```bash
node scripts/package.js
```

This copies `dist/` and `app/config.xml` into the `package/` directory. The `package/` directory is then wrapped into a `.wgt` file by Tizen Studio.

### Step 3: Sign the Package with Tizen Studio

1. Open **Tizen Studio**
2. Connect your target device (TV or emulator) via the **Connection Explorer**
3. In the **Project Explorer**, right-click on your project → **Build and Package** → **Build Release Package**
4. Right-click on the `package/` directory → **Signing** → **Sign with Author Certificate**
5. Select your author certificate and enter its password
6. Tizen Studio exports the signed `.wgt` to the `package/` directory

### Step 4: Deploy to Device

1. In Tizen Studio's **Connection Explorer**, right-click on your device → **Permit to install applications**
2. Right-click on the device → **Install...**
3. Browse to the signed `.wgt` file and install

### Alternative: Sign via CLI (Advanced)

If you prefer command-line signing (requires Tizen SDK tools on `PATH`):

```bash
# Package the .wgt manually
cd package
zip -r ../Phlix-1.0.0.wgt *
cd ..

# Sign with the author certificate
# (sdb and pkgcmd are part of the Tizen SDK)
sdb devices  # verify device connection
sdb push author.p12 /tmp/author.p12
sdb shell "python /usr/bin/pkgcmd -t wgt -p /tmp/Phlix-1.0.0.wgt -s /tmp/author.p12 -c <password>"
```

The exact CLI signing flow depends on your Tizen SDK version and whether you use the device's shell or a host PC. The Tizen Studio GUI workflow is recommended for most developers.

## CI Signing Configuration (When Ready)

Wiring CI to sign builds requires storing private key material in secrets, similar to the other platforms.

### Option A: CI Wraps Build in Tizen Studio (Linux runner)

Tizen Studio does not ship a standalone CLI signing tool for Linux. If using a Linux CI runner, the most reliable approach is to install Tizen Studio in the CI environment and use the author certificate from a secret:

```yaml
- name: Install Tizen Studio
  run: |
    curl -L https://storage.googleapis.com/tizen-sdk/latest/tizen-sdk.bin -o tizen-sdk.bin
    chmod +x tizen-sdk.bin
    ./tizen-sdk.bin --accept-license

- name: Setup Certificate
  env:
    AUTHOR_CERT: ${{ secrets.TIZEN_AUTHOR_CERT }}
    AUTHOR_CERT_PASSWORD: ${{ secrets.TIZEN_AUTHOR_CERT_PASSWORD }}
  run: |
    echo "$AUTHOR_CERT" | base64 --decode --output author.p12
    mkdir -p ~/.tizen-sdk-data/keystore
    cp author.p12 ~/.tizen-sdk-data/keystore/author.p12

- name: Build and Package
  run: |
    npm install
    npm run build
    node scripts/package.js

- name: Sign .wgt
  env:
    AUTHOR_CERT_PASSWORD: ${{ secrets.TIZEN_AUTHOR_CERT_PASSWORD }}
  run: |
    export TIZEN_SDK=/path/to/tizen-sdk
    export PATH="$TIZEN_SDK/tools:$TIZEN_SDK/tools/ide:$PATH"
    # Use Tizen CLI to sign (requires the IDE tool)
    $TIZEN_SDK/tools/ide/bin/tizen package -t wgt \
      -s ~/.tizen-sdk-data/keystore/author.p12 \
      --password "$AUTHOR_CERT_PASSWORD" \
      -o ./release package/
```

> **Note**: The Tizen CLI for signing is part of the IDE tools (`tools/ide/bin/`). Non-IDE Tizen SDK installations may not include signing utilities. In that case, a macOS or Windows CI runner with Tizen Studio GUI available may be required.

### Option B: macOS/Windows Runner (Preferred for Signing)

The simplest CI setup is a Windows or macOS runner with Tizen Studio installed:

```yaml
jobs:
  build-and-sign:
    runs-on: windows-latest  # or macos-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Install Tizen Studio
        run: |
          curl -L https://storage.googleapis.com/tizen-sdk/latest/tizen-sdk.bin -o tizen-sdk.bin
          ./tizen-sdk.bin --accept-license --no-java-jdk

      - name: Import Certificate
        env:
          TIZEN_AUTHOR_CERT: ${{ secrets.TIZEN_AUTHOR_CERT }}
          TIZEN_AUTHOR_CERT_PASSWORD: ${{ secrets.TIZEN_AUTHOR_CERT_PASSWORD }}
        run: |
          echo "$TIZEN_AUTHOR_CERT" | base64 --decode --output author.p12
          mkdir "$env:USERPROFILE\tizen-sdk-data\keystore"
          Copy-Item author.p12 "$env:USERPROFILE\tizen-sdk-data\keystore\author.p12"

      - name: Build
        run: npm install && npm run build && node scripts/package.js

      - name: Sign and Package
        env:
          TIZEN_AUTHOR_CERT_PASSWORD: ${{ secrets.TIZEN_AUTHOR_CERT_PASSWORD }}
        run: |
          # Use Tizen Studio CLI to package and sign
          & "C:\tizen-sdk\tools\ide\bin\tizen" package -t wgt `
            -s "$env:USERPROFILE\tizen-sdk-data\keystore\author.p12" `
            --password "$env:TIZEN_AUTHOR_CERT_PASSWORD" `
            -o release package/
```

Store these as **GitHub Actions secrets**:
- `TIZEN_AUTHOR_CERT` — base64-encoded `.p12` author certificate
- `TIZEN_AUTHOR_CERT_PASSWORD` — certificate password
- `SAMSUNG_CERT` — base64-encoded Samsung distributor certificate (if submitting to Samsung store)
- `SAMSUNG_CERT_PASSWORD` — Samsung certificate password

## Known Issues

- **"Author certificate not found"**: The author certificate must be registered in Tizen Studio's Certificate Manager on the machine doing the signing. For CI, the certificate file must be placed in the `tizen-sdk-data/keystore/` directory (or equivalent).
- **"Device not authorized"**: The TV must be in Developer Mode and have "Permit to install applications" enabled from Tizen Studio before a `.wgt` can be installed on it.
- **Samsung certificate not accepted**: If submitting to the Samsung Apps TV Store, both a Tizen author certificate AND a Samsung distributor certificate are required. Using only the Tizen author certificate works for physical devices but not store submission.
- **Certificate password expired or changed**: If you change the author certificate password, any previously signed packages cannot be re-signed — you must rebuild and sign again.
- **Linux CI limitations**: Tizen Studio's full IDE and signing toolchain requires a graphical environment. Headless Linux CI may not support GUI-based signing. Use a Windows or macOS runner for signed builds in CI, or investigate Docker-based Tizen SDK images.
